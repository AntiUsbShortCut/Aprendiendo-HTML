CODIFICACION HTML (JUEGOS DE CARACTERES)

Para mostrar una página HTML correctamente, un navegador web debe saber qué juego de caracteres utilizar.

1.- De ASCII a UTF-8

ASCII fue el primer estándar de codificación de caracteres. ASCII definió 128 caracteres diferentes que podrían
usarse en Internet: números (0-9), letras en inglés (AZ) y algunos caracteres especiales como! $ + - () @ <>.
ISO-8859-1 era el juego de caracteres predeterminado para HTML 4. Este juego de caracteres admitía 256 códigos
de caracteres diferentes. HTML 4 también es compatible con UTF-8.
ANSI (Windows-1252) fue el juego de caracteres original de Windows. ANSI es idéntico a ISO-8859-1, excepto que
ANSI tiene 32 caracteres adicionales.
El juego de caracteres predeterminado para HTML5 es UTF-8, que cubre casi todos los caracteres y símbolos del mundo.

2.- El atributo de juego de caracteres HTML

Para mostrar una página HTML correctamente, un navegador web debe conocer el juego de caracteres utilizado en la página.
Esto se especifica en la etiqueta meta

3.-Diferencias entre conjuntos de caracteres

El conjunto de caracteres ANSI (Windows-1252)

ANSI es idéntico a ASCII para los valores de 0 a 127.
ANSI tiene un conjunto de caracteres patentado para los valores de 128 a 159.
ANSI es idéntico a UTF-8 para los valores de 160 a 255.

El conjunto de caracteres ISO-8859-1

ISO-8859-1 es idéntico a ASCII para los valores de 0 a 127.
ISO-8859-1 no usa los valores de 128 a 159.
ISO-8859-1 es idéntico a UTF-8 para los valores de 160 a 255.

El conjunto de caracteres UTF-8

UTF-8 es idéntico a ASCII para los valores de 0 a 127.
UTF-8 no usa los valores de 128 a 159. 
UTF-8 es idéntico a ANSI y 8859-1 para los valores de 160 a 255.
UTF-8 continúa desde el valor 256 con más de 10 000 caracteres diferentes.
